Prototype change log

This summarises the main changes we’ve made in response to feedback.

Version 1a to 1b

Version 1 was our first version prototype.

1a was user tested. 1b was formed as a second iteration of version 1, in responce to user testing.

The table below lists the changes between version 1a and 1b following user feedback.

Summary of change in v1

Reason/source

Changed header to sentence case on online licence page

Grammar correction

Alignment change from two column table list, to left aligned single column list on online licence page

Users had trouble understanding the information on the page

Name and site address added on online licence page

Internal user feedback states this information is useful for site visits

Other contact details link moved up the page, next to associated content on online licence page

Feedback suggests links were not easy to spot in their previous positions

Abstraction point grid ref - title change, and data format change on online licence page

Internal users expected to see SJ or SK grid references

Abstraction amount label change on online licence page

Mirroring the format on the paper licence to make amounts clearer

Hands off flow restriction added to online licence page

Internal users noted this is important information

Changed header to sentence case on terms of use page

Grammar correction

Changed header to sentence case to contact details page

Grammar correction

Changed ‘Business details’ to ‘Abstraction return contact details’ on contact details page

Internal user feedback states that returns have a separate address, no user research has suggested that business address is required

Version 2

Version 2 was our second version prototype.

The table below lists the changes between Version 1 and 2 following user feedback.

Summary of changes in v2

Reason/source

Updated quantity condition in line with example licence

Continuity update

Abstraction point grid ref changed in line with example licence

Continuity update

Date effective from has been added to the licence page

Feedback from internal user is that this can be different from the date of which the licence is obtained

Period of abstraction to seasonal information, rather than date ranges

Feedback from internal users is that this is the current format, as confirmed by the example licence, updated in line with example licence

Added map pages with responsive maps, linked to from a maps list on terms of use page

To obtain user feedback on map user needs

Removed ‘change’ links on contact page

These links are currently not linked up to a function, so should be removed

Billing contact details have been added on the contact page

Internal user feedback states that Billing is a further separate set of address and contact details

External user pages have been added for testing. These are to remain the same as internal user view pages for now, but may change based on user need

Anticipating separate feedback from two sets of user groups
Instead of 'Effective from' and 'Expiry date' listed seperatly, 'Licence effective dates: 11 July 2016 - 11 July 2017' To save list space
Map list changed to only include one map, the abstraction point As the other example maps were confusing users, as not current examples. Users will tell us if any maps are missing. In the event it is confirmed only one map is needed, we will change the show and hide to a direct link.
Primary address changed to 'Site contact details', Licence holder details, changed to 'Registered licence holder details' Changed following feedback from a subject matter expert
Contact details link on the licence page moved to next to name, and changed from 'Other contact details' to 'Contact details' Changed following internal user confusion and feedback
Terms page updated with an accordion, rather than a series of show and hide links Bugs were being reported with the use of multiple show and hides, and this version is much more mobile friendly regarding the clickable area
